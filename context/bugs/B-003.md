# B-003: production landing page shows a developer placeholder "Route stub: / marketing"
Severity: S3 minor
Found by: TR-009 while testing T-009 (design review, landing screenshot)
Status: verified-closed

## Repro steps
1. Open https://amityx.pages.dev/ (the public root, the first thing a prospective hub
   owner sees).
2. Read the page.

## Expected vs actual
Expected: a clean, launch-appropriate landing (product name + value line + a single
"Start your hub" primary CTA).
Actual: the page renders a leftover scaffold placeholder — an eyebrow reading
**"ROUTE STUB: / MARKETING"** (`src/pages/marketing/Landing.tsx`, a T-003 stub whose
comment said "Real copy/design lands with T-005" — but T-005 was the tenancy schema, so
the stub was never replaced and shipped to production). P.9 rule 10: "if it needs
explaining, redesign it" — a debug label on the front door is the opposite of that.

## Evidence
- Screenshot: `app/web/e2e/screens/landing-375.png` (+ `-768/-1280/-dark`).
- Source: `Landing.tsx` line 13-15 renders `Route stub: <code>/</code> marketing`.

## Fix design
Remove the placeholder eyebrow `<p>` so the landing is a clean minimal front door
(logo + tagline + one primary CTA — all already present and on-brand). No other change;
a fuller marketing page is a separate future task, out of scope for this cycle.
