## Spec
Supabase Auth (email+password, plus email OTP as a passwordless option); custom SMTP = Google
Workspace help@agapaycare.com configured in the Supabase dashboard (env NAMES only, never in app
code). Signup wizard, each step one job (P.9): (1) owner account + email verify → (2) hub creation
(name → auto-slug, collision-checked) → (3) program-template picker (art, swim, karate, day care,
boot camp, open-play, camp; multi-select with sensible pre-checks) → (4) first class schedule →
(5) optional staff invites by email. Wizard progress persists so a drop-off resumes where it left
off (protects the ≤15-min criterion). On success, land on a populated hub dashboard. CRM handoff:
signup atomically creates the hub's crm_hub_profiles row (onboarding_stage seeded). Session persists
+ auto-refreshes (supabase-js). Error behavior (P.9 rule 8 — never a dead end): duplicate email →
"That email already has an account. Sign in?"; weak password → inline rule; slug collision →
auto-suffix + editable; SMTP send failure → retry + surface the verification path. Staff invitees
are provisioned WITHOUT billing/settings access (enforced by T-005 RLS).
