## Spec
Zero during-class burden (D-009). /app mobile-first: Today (sessions now/next) → session roster →
**kiosk mode** (hub tablet: parent taps child's name+photo to check in/out, PIN-less, staff-lockable)
+ one-tap staff fallback (timestamped, offline-tolerant optimistic UI) → per-child daily note
(text, quick-tags — written AFTER class, ≤3 taps). Children & guardians CRUD; enrollment
management; booking_requests inbox (accept → auto-enroll, D-009); capacity indicators.

## Spec — mechanics (clarifying, not rescoping)
Nav = /app bottom tabs, max 4 (P.9 rule 6): Today · Roster · Requests · More. Kiosk mode is a
full-screen lock launched from Today; a parent taps their child's name+photo tile to Check in /
Check out (PIN-less); exiting kiosk requires a staff long-press/PIN so parents can't reach admin.
One-tap staff fallback = optimistic UI: the tap commits locally, queues, and syncs on reconnect;
each attendance write is idempotent by (session, child, action) so a double-tap or replay can't
duplicate. Daily note is written AFTER class: free text + quick-tags, ≤3 taps, autosaves.
Vocabulary per P.9 (Classes/Activities/Roster/Notes/Requests — never the schema names). Requests
inbox: accept → auto-enroll into the chosen program/session (respects capacity → offer waitlist if
full); decline is undoable. Empty states teach ("Add your first class"). Edge cases: child not yet
in the system at the door → quick-add tile; session at capacity → explicit waitlist path, never a
silent overbook.
